全同 UWFBG 重叠畸变解调项目当前技术状态总结
日期：2026-03-17
用途：作为论文写作阶段的技术状态参考，重点对齐“复杂条件下的光谱畸变解调”这一小论文主线。

============================================================
一、文档目的与建议定位
============================================================

这份总结不是新的实验计划，而是对当前已经完成的技术工作、已验证结论、可写入论文的主线、以及不宜过度宣称的部分进行系统整理。

结合现有论文计划，当前最合理的写作定位是：

1. 小论文主线
   以“复杂条件下的全同 UWFBG 重叠畸变光谱解调”为核心问题，
   重点围绕可控仿真平台、hardest case 场景、强 baseline CNN、以及 Tail-aware loss 的尾部优化效果展开。

2. 大论文嵌入方式
   将这部分工作作为“长距离全同 UWFBG 阵列 OFDR 系统鲁棒解调与稳定性评估”中的一个关键子问题：
   即，复杂工况下谱形失稳会导致解调性能下降，而本文先在可控仿真平台中系统研究其中最典型的一类问题：结构性畸变导致的目标光栅解调误差。

3. 当前最稳妥的技术叙事
   不是宣称提出了一个大幅领先的新 backbone，
   而是：
   - 构建了一个面向全同 UWFBG 结构性畸变的可控仿真平台；
   - 证明了 CNN 相比传统方法更适合该问题；
   - 识别出 hardest case 中误差主要来自尾部困难样本；
   - 在此基础上提出 Tail-aware loss，并通过多 seed 结果证明其稳定的小幅改善。


============================================================
二、当前任务的准确定义与边界
============================================================

这部分非常重要，论文中必须表述准确，否则容易把任务边界写错。

1. 当前任务是什么
   当前项目的核心任务是：
   从一段包含目标峰及邻峰耦合影响的一维局部 OFDR/UWFBG 反射谱中，回归目标光栅的中心波长漂移量 delta_lambda_target，或其等价温度/应变表征量。

2. 当前任务不是什么
   当前任务不是“同时独立解调 5 个光栅各自的温度”。
   当前数据、标签和模型的定义，都是围绕“指定目标光栅”的单目标稳健解调来设计的。

3. 因而，当前工作解决的关键问题是
   在邻峰相对位移、线宽变化、局部伪影、尖峰干扰、baseline/ripple 等共同作用下，
   即使目标峰的局部形态已经发生明显结构性畸变，
   模型仍能尽量准确地恢复目标光栅的真实中心漂移。

4. 推荐论文中的准确表述
   可写为：
   “本文关注全同 UWFBG 重叠条件下目标光栅的鲁棒解调问题，而非完整多光栅参数联合反演问题。”


============================================================
三、仿真平台与困难因素建模状态
============================================================

当前项目已经形成了较完整的、可控的困难场景仿真平台。这是论文的重要基础贡献之一。

1. 仿真对象
   每个样本不是单一理想峰，而是一个局部组合谱，通常包含：
   - target peak
   - 左邻峰
   - 右邻峰
   - 以及背景项和局部扰动项

2. 当前已经明确建模的困难因素
   - neighbor shift：邻峰相对位置偏移，改变局部重叠关系
   - linewidth variation：峰形展宽/收窄，改变局部谱形判别边界
   - target-nearby artifact：目标峰附近局部伪影、肩峰、小畸变
   - spike disturbance：窄而尖的局部突刺
   - baseline drift / ripple：缓变背景和纹波扰动

3. 当前 hardest case 的定义
   hardest case 目前固定为：
   - shift004 + linewidth_l3

   它是目前所有方法增强和多 seed 比较的主场景。
   这一定义已经比较稳定，后续论文正文中建议作为主困难集固定使用。

4. 当前仿真平台的价值
   - 困难因素是可控、可分离、可组合的
   - 可以单独分析某一种畸变因素的影响
   - 可以构造 hardest case 作为统一 benchmark
   - 非常适合支撑“方法比较 + 误差来源分析 + 鲁棒性分析”的完整论文链条


============================================================
四、评价体系与公平对照原则
============================================================

当前实验体系已经比较规范，适合直接写入论文方法与实验设置部分。

1. 主要评价指标
   - RMSE：整体误差，且对大误差更敏感
   - MAE：平均误差水平
   - P95：后 5% 困难样本之前的误差上界
   - P99：极端困难样本误差
   - R2：回归拟合优度

2. 当前实验的核心判断标准
   本项目当前最关键的鲁棒性指标不是仅看 MAE 或 RMSE，
   而是要同时看：
   - 平均指标：RMSE、MAE
   - 尾部指标：P95、P99

3. 为什么 P95/P99 特别重要
   当前 hardest case 的问题不是“整体平均精度很差”，而是“少量结构性困难样本拉高了尾部误差”。
   因此，如果一个方法只改善均值但不改善 P95/P99，或者只改善单 seed 而缺乏稳定性，都不能算真正解决了核心问题。

4. 当前已经遵守的公平比较原则
   在近期主要对比中，基本都遵守了以下规则：
   - 相同数据集
   - 相同 epochs / patience
   - 相同 batch size
   - 相同 optimizer / scheduler
   - 尽量保持 backbone 不变，只最小改动新增变量
   - 通过 5-seed 验证稳定性

============================================================
七、Tail-aware loss：当前最清晰、最可写的主方法
============================================================

------------------------------------------------------------
7.1 方法动机
------------------------------------------------------------

hardest case 下，baseline CNN 的平均误差已经不算高，但 P95/P99 仍然偏高。
这表明主要问题不在一般样本，而在少量结构性困难样本。

因此当前最合理的改进方向不是继续堆复杂 backbone，而是直接修改优化目标，让模型在训练时更关注超阈值的大误差样本。

------------------------------------------------------------
7.2 当前实现形式
------------------------------------------------------------

Tail-aware loss 形式为：

L_i = |e_i| + lambda_tail * max(0, |e_i| - tau)^2

其中：
   - e_i = prediction_i - target_i
   - tau 默认使用 0.01
   - lambda_tail 默认使用 3.0

它的含义是：
   当样本误差没有超过阈值 tau 时，基本保持普通 L1 行为；
   当误差超过阈值 tau 时，附加一个二次增强惩罚，
   从而让训练更关注真正构成 P95/P99 的尾部困难样本。

------------------------------------------------------------
7.3 单次 seed 结果
------------------------------------------------------------

hardest case, seed=42:

Baseline CNN:
   - RMSE = 0.01388817
   - MAE  = 0.01070953
   - P95  = 0.02783730
   - P99  = 0.03923561
   - R2   = 0.99768269

CNN + Tail-aware:
   - RMSE = 0.01323813
   - MAE  = 0.01017639
   - P95  = 0.02557017
   - P99  = 0.03709696
   - R2   = 0.99789453

结论：
   在这个单次 seed 中，Tail-aware 对所有主要指标均实现了改善。

------------------------------------------------------------
7.4 Tail-aware + Hard weighting
------------------------------------------------------------

1. 当前实现
   训练阶段额外加入 batch 内基于当前误差的轻量 hard weighting：

   w_i = 1 + alpha * I(|e_i| > tau_hard)

   并用 w_i 对 per-sample loss 加权。

2. 动机
   它比 Tail-aware 更激进，试图进一步强化困难样本。

3. 单次 seed 结果
   CNN + Tail-aware + Hard weighting:
   - RMSE = 0.01447156
   - MAE  = 0.01117076
   - P95  = 0.02852043
   - P99  = 0.04169144
   - R2   = 0.99748391

4. 单次判断
   单次 seed 中它反而退化，说明激进 hard weighting 的风险很高。

九、最近的尾部优化补充探索及结论
============================================================

在 Tail-aware 主线之外，还做过若干更进一步的探索，主要围绕 G raw+d1 路线和更温和的 tail/hard 组合。

1. G + Tail-aware
   在单次结果中曾出现：
   - P99 略有回拉
   - 但 P95 并未继续优于 G 本身

2. G + two-stage (MSE -> Tail-aware)
   当前参数下未显示出额外优势。

3. G + mild tail/hard 小网格
   曾出现若干单次 seed 看起来较亮眼的组合，例如同时获得较低 P95 和较低 P99，
   但在多 seed 下没有形成“稳定同时优于 baseline 与 G”的结论。

4. 当前最重要的实际结论
   P95 和 P99 在当前任务中存在明显 trade-off。
   某些组合可以更好压低 P95，
   但不一定能同步稳定压低 P99，或不一定能保持 RMSE/MAE 不退化。

5. 因此对论文主线的影响
   这些探索更适合作为“后续工作或补充实验”，
   不宜抢占 Tail-aware 主线的位置。


============================================================
十、当前最成立的技术结论
============================================================

截至目前，可以比较有把握地成立的结论包括：

1. 任务层面
   当前研究的是“结构性畸变条件下目标光栅的稳健解调”，而不是多光栅同时独立温度反演。

2. 场景层面
   neighbor shift、linewidth variation、artifact、spike、baseline/ripple 的叠加，会造成明显的谱形形貌变化；
   hardest case 确实是当前任务中最有代表性的困难场景。

3. 基线层面
   CNN 已经是强 baseline；
   传统方法在结构性畸变场景下明显受限。

4. 方法层面
   当前主瓶颈不在于 backbone 过弱，而在于尾部困难样本没有被合理优化。

5. 结构探索层面
   - BiLSTM 不是当前最有效路线
   - NADNet 当前版本不成立为主线
   - 三通道导数输入 raw+d1+d2 为负收益
   - raw+d1 有一定启发，但不构成稳定主方案

6. 当前最清晰有效的方法结论
   Tail-aware loss 是当前最值得写入论文的核心方法改进。

7. 稳定性层面
   多 seed 结果已经支持这样一个保守而成立的判断：
   Tail-aware 能带来稳定但有限的小幅改善，
   而更激进的 hard weighting 仍存在稳定性问题。


============================================================
十一、论文中可以安全宣称的贡献点
============================================================

建议采用保守、可支撑、不过度夸大的表述。

可以安全写成：

1. 构建了面向全同 UWFBG 结构性重叠畸变解调问题的可控仿真平台，
   能系统刻画邻峰漂移、线宽变化及局部扰动对局部组合谱的影响。

2. 在该平台上证明了传统解调方法在复杂结构性畸变场景下明显受限，
   而 CNN 可作为有效强 baseline。

3. 通过 hardest case 分析发现，当前误差瓶颈主要来自尾部困难样本，而不仅仅是平均误差水平。

4. 针对尾部困难样本提出 Tail-aware loss，
   通过对超阈值误差施加增强惩罚，提高复杂工况下的解调稳定性。

5. 通过 5-seed 实验表明，Tail-aware loss 能带来稳定但有限的小幅改善；
   更激进的 hard weighting 虽有进一步压低尾部误差的潜力，但稳定性不足。


============================================================
十二、论文中不建议过度宣称的点
============================================================

1. 不建议写成“提出了显著优于所有现有方法的新深度网络”
   因为目前并没有一个复杂结构模型形成全面稳定优势。

2. 不建议写成“解决了多光栅独立解调问题”
   因为当前任务不是这个定义。

3. 不建议写成“hard weighting 已被证明优于 Tail-aware”
   目前只能说它有潜力，但稳定性不够。

4. 不建议过度强调导数输入或 NAD 路线
   因为它们都没有形成当前阶段最可信的正结果。

5. 不建议把单次最优 seed 的结果当成主要结论
   论文必须以多 seed 结果为主证据。

十三、当前最推荐的论文结果组织方式
============================================================

建议沿着下面这条逻辑线组织结果章节：

1. 先讲仿真平台和困难因素
   - 解释为什么是结构性畸变问题
   - 解释 hardest case 的定义

2. 再讲 baseline CNN
   - 输入、输出、指标
   - 训练收敛性
   - baseline 是可信且足够强的

3. 再讲传统方法 vs CNN
   - 建立“为什么需要学习式解调”

4. 再讲结构增强尝试为什么没有成为主线
   - LightSE / BiLSTM / NAD / raw+d1 等探索的结论
   - 说明问题主矛盾不在 backbone

5. 最后推出 Tail-aware loss
   - hardest case 的尾部误差问题
   - 损失设计动机
   - 单次结果
   - 多 seed 结果
   - 稳定性分析

6. 章节收束
   - 强调 Tail-aware 是“轻量、针对性、稳定的小幅改进”
   - 不是大幅突破，但技术故事完整、证据链清楚
